# PSD 剪切蒙版在 Godot 中的实现说明

这份文档解释本插件如何把 Photoshop 里的“剪切蒙版”导入成 Godot 场景树结构。写法尽量照顾 Godot 新人：先说效果，再说代码为什么这么做。

## 1. 什么是剪切蒙版

在 Photoshop 里，剪切蒙版的意思是：

> 上面的图层只显示在下面那个“基底图层”有内容的地方。

举个例子：

- 下面有一个圆形图层。
- 上面有一张很大的图片。
- 如果这张图片是圆形图层的剪切蒙版，那么图片只会显示在圆形范围里。

可以把下面的图层理解成一个“窗口”或“形状遮罩”，上面的剪切图层只能透过这个窗口显示。

## 2. PSD 文件里怎么表示剪切蒙版

PSD 的每个图层记录里都有一个 `clipping` 字节。

插件解析图层记录时会读取它：

```gdscript
var clipping := f.get_8()
rec["is_clipping_mask"] = clipping != 0
```

然后在 `LayerData` 里保存成：

```gdscript
var is_clipping_mask: bool = false
```

也就是说：

- `is_clipping_mask == false`：普通图层，可能会成为基底图层。
- `is_clipping_mask == true`：剪切图层，应该被它下面最近的基底图层裁剪。

## 3. Photoshop 顺序和 Godot 顺序不同

Photoshop 的图层面板通常是“上面的图层在列表前面”。

Godot 的场景树绘制通常是“先添加的节点在下面，后添加的节点在上面”。

所以插件会先把图层顺序反过来，变成更适合 Godot 绘制的顺序：

```gdscript
func _drawing_order(source: Array) -> Array:
	var ordered := source.duplicate()
	ordered.reverse()
	return ordered
```

这样底部图层会先进入场景树，上面的图层后进入场景树，视觉层级就更接近 Photoshop。

## 4. Godot 里怎么模拟剪切蒙版

Godot 没有一个和 Photoshop 剪切蒙版完全一样的 PSD 图层系统，所以插件使用 Godot 的父子节点裁剪能力来模拟。

核心思路是：

1. 找到一个普通图层，作为“基底图层”。
2. 看它后面有没有连续的 `is_clipping_mask == true` 图层。
3. 如果有，就把这些剪切图层变成基底图层的子节点。
4. 给基底图层开启 `clip_children`，让子节点被父节点裁剪。

对应代码在 `addons/psd_importer/scene_builder.gd` 的 `_add_layers_with_clipping()`：

```gdscript
if not clip_layers.is_empty():
	_enable_child_clipping(node)
	var base_origin := Vector2(float(layer.left), float(layer.top))
	for clip_layer in clip_layers:
		var clip_node := _build_node(clip_layer, base_origin)
		if clip_node:
			node.add_child(clip_node)
```

基底节点开启裁剪：

```gdscript
func _enable_child_clipping(node: Node) -> void:
	if node is CanvasItem:
		node.clip_children = CanvasItem.CLIP_CHILDREN_AND_DRAW
	if node is Control and node.size == Vector2.ZERO:
		node.size = _node_fallback_size(node)
```

这里最关键的是：

```gdscript
node.clip_children = CanvasItem.CLIP_CHILDREN_AND_DRAW
```

它表示：父节点自己正常绘制，同时裁剪它的子节点。

## 5. 场景树会变成什么样

假设 Photoshop 中有这些图层：

```text
图片纹理 2  （剪切蒙版）
图片纹理 1  （剪切蒙版）
圆形底图    （基底图层）
背景
```

导入到 Godot 后，结构会接近这样：

```text
PSD_Scene
├─ 背景
└─ 圆形底图
   ├─ 图片纹理 1
   └─ 图片纹理 2
```

其中 `圆形底图` 会开启：

```gdscript
clip_children = CanvasItem.CLIP_CHILDREN_AND_DRAW
```

这样 `图片纹理 1` 和 `图片纹理 2` 就会被 `圆形底图` 裁剪。

## 6. 为什么要重新计算坐标

当一个图层变成另一个节点的子节点时，它的位置就不再是相对于整个 PSD 画布，而是相对于父节点。

例如：

- 基底图层在画布位置 `(100, 200)`。
- 剪切图层在画布位置 `(130, 240)`。

如果剪切图层变成基底图层的子节点，它的新位置应该是：

```text
(130, 240) - (100, 200) = (30, 40)
```

所以插件会把基底图层的位置作为 `parent_origin` 传进去：

```gdscript
var base_origin := Vector2(float(layer.left), float(layer.top))
var clip_node := _build_node(clip_layer, base_origin)
```

最终位置计算在 `_apply_position()` 里：

```gdscript
var pos := Vector2(float(layer.left), float(layer.top)) - parent_origin
```

如果不做这一步，剪切图层一旦挂到基底图层下面，就会整体偏移，位置看起来会和 Photoshop 不一致。

## 7. 画布外内容为什么也会被裁剪

Photoshop 中，超出画布外的内容通常不可见。

但在 Godot 编辑器里，如果节点超出根节点范围，默认仍然可能被看到。这会导致一些“在 PS 里看不见、在 Godot 里却露出来”的内容。

所以插件在根节点是 `Control` 时，会设置 PSD 画布大小，并开启根节点裁剪：

```gdscript
root.size = canvas_size
root.clip_contents = true
```

这样超出 PSD 画布范围的内容不会在编辑器里露出来。

## 8. 当前实现的适用范围

这个实现是“用 Godot 节点系统模拟 Photoshop 剪切蒙版”，不是完整复刻 Photoshop 渲染引擎。

目前比较可靠的情况：

- 普通图片图层被基底图层裁剪。
- 文字或图片作为剪切图层挂到基底图层下面。
- 多个连续剪切图层挂到同一个基底图层下面。
- 超出 PSD 画布的内容被根节点裁剪。

可能仍然有差异的情况：

- Photoshop 特殊混合模式。
- 复杂图层样式参与剪切后的最终效果。
- 调整图层和剪切蒙版组合。
- 非 `Control` 根节点下的画布矩形裁剪。

## 9. 一句话总结

插件的剪切蒙版实现方式是：

> 读取 PSD 图层的 `clipping` 标记，把剪切图层变成基底图层的子节点，再用 Godot 的 `clip_children` 让基底图层裁剪它们。
